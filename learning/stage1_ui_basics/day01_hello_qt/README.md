### 第1步：包含头文件
```cpp
#include <QApplication>
#include <QWidget>
#include <QPushButton>
#include <QDebug>
```

| 头文件 | 作用 | 类比 |
|--------|------|------|
| `QApplication` | 应用程序管理类 | 相当于程序的"入口大门" |
| `QWidget` | 基础窗口类 | 相当于一个"空白的窗口" |
| `QPushButton` | 按钮控件 | 相当于一个"可点击的按钮" |
| `QDebug` | 调试输出 | 相当于C++的`cout` |

---

### 第2步：main函数
```cpp
int main(int argc, char *argv[])
{
```
标准的C++入口函数，`argc`和`argv`用于接收命令行参数。

---

### 第3步：创建应用程序对象 ⭐核心
```cpp
    // 创建应用程序对象
    // 每个Qt程序必须有且只有一个QApplication对象
    // 它负责管理应用程序的全局资源和事件循环
    QApplication app(argc, argv);
```

**这是最重要的对象！**

- **必须性**：每个Qt程序必须有且只有一个`QApplication`
- **作用**：
  1. 管理全局资源（字体、图标等）
  2. 处理命令行参数
  3. **管理事件循环**（等待用户操作）

**类比**：就像一家餐厅的前台，负责接待所有顾客（事件）。

---

### 第4步：创建窗口
```cpp
    // 创建主窗口
    QWidget window;
    
    // 设置窗口标题
    window.setWindowTitle("Hello Qt - 我的第一个Qt程序");
    
    // 设置窗口大小 (宽度, 高度)
    window.resize(800, 600);
```

| 方法 | 作用 |
|------|------|
| `QWidget window;` | 创建一个空白窗口 |
| `setWindowTitle()` | 设置窗口标题栏文字 |
| `resize(宽, 高)` | 设置窗口大小（像素） |

**注意**：此时窗口还未显示，只是创建好了。

---

### 第5步：添加按钮
```cpp
    // 创建一个按钮
    QPushButton *button = new QPushButton("点击我", &window);
    button->setGeometry(350, 250, 100, 40);  // 设置按钮位置和大小
```

| 参数 | 含义 |
|------|------|
| `"点击我"` | 按钮上显示的文字 |
| `&window` | 按钮的父窗口（重要！内存自动管理） |
| `setGeometry(x, y, 宽, 高)` | 设置位置和大小 |

**`&window`的作用**：
- 指定按钮属于哪个窗口
- **内存管理**：当`window`被销毁时，按钮会自动销毁
- 这就是Qt的父子对象机制！

---

### 第6步：显示窗口
```cpp
    // 显示窗口
    window.show();
```
**`show()`方法**：让窗口真正显示在屏幕上。

**注意**：之前只是创建，这里才真正显示出来。

---

### 第7步：调试输出
```cpp
    // 输出调试信息
    qDebug() << "程序已启动，窗口已显示";
```
`qDebug()`是Qt的调试输出，类似于：
- C++的 `std::cout`
- C的 `printf`

**特点**：
- 自动添加换行
- 可以输出Qt类型（如QString）
- 发布模式下自动禁用

---

### 第8步：进入事件循环 ⭐核心
```cpp
    // 进入事件循环，等待用户交互
    // exec()会一直运行，直到程序退出
    return app.exec();
}
```

**这是程序的核心！**

`app.exec()`的作用：
1. **阻塞**：程序停在这里，等待用户操作
2. **监听事件**：鼠标点击、键盘输入、窗口关闭等
3. **分发事件**：将事件分发给对应的控件处理
4. **直到退出**：当最后一个窗口关闭时，`exec()`返回

**类比**：就像餐厅前台开始工作，等待顾客上门，直到打烊。

## 程序执行流程图

```
开始
  │
  ▼

  │
  ▼

  ▼

  ▼
显示窗口 window.show()
  │
  ▼
输出调试信息
  │
  ▼
进入事件循环 app.exec() ←──── 等待用户操作
  │                              │
  │                              ▼
  │                    用户点击按钮/关闭窗口
  │                              │
  │                              ▼
  │                    处理事件，继续等待
  │                              │
  └──────────────────────────────┘
  │
  ▼
程序结束（窗口关闭后）
```

## 关键概念总结

| 概念 | 说明 | 必须掌握 |
|------|------|----------|
| **QApplication** | 应用程序管理，必须有且只有一个 | ⭐⭐⭐ |
| **QWidget** | 基础窗口，所有UI控件的基类 | ⭐⭐⭐ |
| **父子对象** | `new QPushButton(..., &window)` 内存自动管理 | ⭐⭐⭐ |
| **事件循环** | `exec()` 让程序等待用户交互 | ⭐⭐⭐ |
| **show()** | 真正显示窗口 | ⭐⭐⭐ |
